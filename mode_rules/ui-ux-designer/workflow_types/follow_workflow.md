# Follow Workflow - UI/UX Designer Rules

## Overview
These rules extend the core UI/UX Designer rules specifically for "Follow" workflow projects, where close collaboration with the user is expected throughout the design process. In Follow workflows, the UI/UX Designer must maintain continuous engagement with the user, seeking frequent validation and feedback on design decisions.

## Follow Workflow-Specific Design Process

### 1. Iterative Design Approach
- **Required Approach**:
  - Break the design process into multiple, smaller iterations
  - Present design work to the user after each iteration for validation
  - Refine designs based on user feedback
  - Document the evolution of designs over time
  - Create design artifacts at increasing levels of fidelity
  - Plan for multiple feedback cycles

### 2. User Engagement Strategy
- **Required Approach**:
  - Establish clear checkpoints for user feedback on designs
  - Prepare specific design questions for each checkpoint
  - Present design options with clear pros and cons
  - Document user feedback on design decisions
  - Highlight areas where user feedback led to design changes
  - Create comparison views to show design evolution

### 3. Design Presentation
- **Required Approach**:
  - Present designs in context of user scenarios
  - Explain design rationales clearly
  - Use appropriate fidelity for each stage (wireframes, mockups, prototypes)
  - Provide visual aids to illustrate design concepts
  - Explicitly seek user approval for key design decisions
  - Document the reasoning behind design choices

### 4. Prototype-Driven Design
- **Required Approach**:
  - Create interactive prototypes early in the process
  - Use prototypes to validate design decisions with the user
  - Document feedback on prototypes
  - Refine designs based on prototype feedback
  - Increase prototype fidelity with each iteration
  - Focus on validating critical user flows

## Follow Workflow Design Document Adaptations
For Follow workflow projects, adapt the standard design documentation in these ways:

### 1. Living Design Document Approach
- Clearly mark design documents as "Living Documents"
- Include a version history section at the beginning
- Date and timestamp all design updates
- Highlight recent design changes
- Track the evolution of key design decisions
- Maintain a visual history of design iterations

### 2. Feedback Integration Section
- Add a dedicated section for user feedback on designs
- Document how feedback was incorporated into designs
- Track feedback that was deferred or not incorporated (with reasons)
- Link feedback to specific design changes
- Include before/after comparisons where appropriate
- Categorize feedback by type and priority

### 3. Decision Log
- Maintain a running log of key design decisions
- Document the context of each design decision
- Note who was involved in each decision (especially user input)
- Record alternatives that were considered
- Document the rationale for the final design decision
- Include visual references for significant decisions

### 4. Open Questions Section
- Maintain a prominent "Open Design Questions" section
- Assign ownership to each open design question
- Track the status of each question
- Document when and how questions were resolved
- Highlight questions requiring user input
- Prioritize questions by impact on the design

## Follow Workflow User Interaction Guidelines
When interacting with users in a Follow workflow, adhere to these guidelines:

### 1. Preparation
- Review all previous design discussions before each new session
- Prepare a structured agenda for each design review
- Create visual aids to explain design concepts
- Anticipate potential questions or concerns about the designs
- Prepare multiple design options for key decisions
- Create side-by-side comparisons of alternatives

### 2. During Interaction
- Begin by summarizing previous design decisions
- Present design information in digestible chunks
- Use visual aids to explain design concepts
- Avoid design jargon or explain it clearly
- Confirm understanding by restating key design decisions
- End with clear next steps and expectations
- Capture feedback in real-time

### 3. Follow-Up
- Send a summary of the design discussion within 24 hours
- Highlight key design decisions made
- List design questions with owners and deadlines
- Include any updated design documentation
- Provide links to updated prototypes or mockups
- Set expectations for the next design iteration

### 4. Documentation
- Update all relevant design documentation promptly after each interaction
- Highlight changes made based on user feedback
- Maintain a chronological record of all design decisions
- Ensure all team members have access to the latest design documentation
- Version control all design documents and assets
- Create a clear trail of design evolution

## Follow Workflow Design Questions
Use these additional questions when designing in a Follow workflow:

1. **Feedback Process Questions**:
   - "How would you prefer to review and provide feedback on the designs?"
   - "What level of design detail is most helpful for you when reviewing options?"
   - "How frequently would you like to review design iterations?"
   - "Are there specific aspects of the design you're most interested in being involved with?"

2. **Decision-Making Questions**:
   - "Who needs to be involved in key design decisions?"
   - "What factors are most important to you when evaluating design options?"
   - "How quickly can we expect decisions on design questions that arise?"
   - "Are there any existing design preferences or constraints we should be aware of?"

3. **Expectation-Setting Questions**:
   - "What does a successful design look like from your perspective?"
   - "What concerns do you have about the design process?"
   - "Are there any specific design qualities that are particularly important to you?"
   - "What level of design polish do you expect at each stage of the process?"

Remember that in Follow workflows, the design process is collaborative and iterative. Plan for multiple touchpoints with the user and maintain meticulous documentation of all design decisions and their evolution over time.
