# YOLO Workflow - Product Manager Rules

## Overview
These rules extend the core Product Manager rules specifically for "YOLO" (You Only Live Once) workflow projects, where rapid development with minimal user interaction is expected. In YOLO workflows, the Product Manager must make more autonomous decisions, focus on efficiency, and prioritize speed while maintaining quality.

## YOLO Workflow-Specific Requirements

### 1. Accelerated Requirements Gathering
- **Required Approach**:
  - Conduct a single, comprehensive requirements gathering session
  - Focus on capturing essential requirements only
  - Make reasonable assumptions where information is missing
  - Document all assumptions explicitly
  - Prioritize ruthlessly to identify the true MVP

### 2. Autonomous Decision-Making
- **Required Approach**:
  - Make independent decisions on non-critical requirements
  - Document the reasoning behind each decision
  - Identify only the most critical decisions that require user input
  - Batch questions for the user to minimize interruptions

### 3. MVP Definition
- **Required Approach**:
  - Define the absolute minimum viable product
  - Focus on core functionality only
  - Eliminate all nice-to-have features from initial scope
  - Create a clear roadmap for post-MVP enhancements
  - Set explicit boundaries on scope

### 4. Risk Management
- **Required Approach**:
  - Identify high-risk assumptions
  - Create contingency plans for critical assumptions
  - Document potential pivot points
  - Establish clear criteria for when to seek user input

## YOLO Workflow PRD Adaptations
For YOLO workflow projects, adapt the standard PRD in these ways:

### 1. Streamlined Document
- Reduce the PRD to essential sections only
- Focus on clarity and brevity
- Use bullet points and concise language
- Include visual representations where possible

### 2. Assumptions Section
- Create a prominent "Assumptions" section
- Categorize assumptions by risk level
- Document the impact if assumptions prove incorrect
- Identify which assumptions need validation

### 3. Decision Log
- Maintain a concise log of key decisions
- Document the reasoning behind each decision
- Note which decisions were made autonomously
- Highlight decisions that may need user validation later

### 4. Scope Boundaries
- Create an explicit "Out of Scope" section
- Be specific about what is NOT included
- Document features deferred to future iterations
- Set clear expectations about limitations

## YOLO Workflow User Interaction Guidelines
When interacting with users in a YOLO workflow, adhere to these guidelines:

### 1. Initial Engagement
- Set clear expectations about the YOLO approach
- Explain the tradeoffs between speed and user involvement
- Establish boundaries for autonomous decision-making
- Identify critical decision points that will require user input

### 2. Efficient Communication
- Batch questions to minimize interruptions
- Present options with clear recommendations
- Focus on decisions with highest impact
- Provide context and reasoning for each question

### 3. Progress Updates
- Send concise, regular updates
- Focus on key milestones and blockers
- Highlight decisions made and assumptions taken
- Clearly indicate any items requiring user attention

### 4. Documentation
- Keep documentation lean but sufficient
- Focus on documenting decisions and rationales
- Maintain a living list of future enhancements
- Document technical debt incurred for future reference

## YOLO Workflow Requirements Gathering Questions
Use these additional questions when gathering requirements in a YOLO workflow:

1. **Prioritization Questions**:
   - "If we could only deliver three features, which would they be?"
   - "What is the absolute minimum functionality needed for this product to be useful?"
   - "Which features could be deferred to a future iteration with minimal impact?"

2. **Decision Authority Questions**:
   - "Which types of decisions are you comfortable with us making independently?"
   - "What decisions absolutely require your input?"
   - "How should we handle situations where we need a quick decision from you?"

3. **Risk Tolerance Questions**:
   - "How comfortable are you with us making assumptions to move quickly?"
   - "What areas of the product are most critical to get right the first time?"
   - "What level of polish is necessary for the initial release?"

Remember that in YOLO workflows, the focus is on speed and efficiency. Make reasonable assumptions, document them clearly, and focus on delivering the core value proposition as quickly as possible while maintaining a path to future enhancements.
