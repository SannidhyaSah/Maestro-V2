# Follow Workflow - Product Manager Rules

## Overview
These rules extend the core Product Manager rules specifically for "Follow" workflow projects, where close collaboration with the user is expected throughout the development process. In Follow workflows, the Product Manager must maintain continuous engagement with the user, seeking frequent validation and feedback.

## Follow Workflow-Specific Requirements

### 1. Iterative Requirements Gathering
- **Required Approach**:
  - Break requirements gathering into multiple, smaller sessions
  - Present findings back to the user after each session for validation
  - Refine requirements based on user feedback
  - Document the evolution of requirements over time

### 2. User Engagement Strategy
- **Required Approach**:
  - Establish clear checkpoints for user feedback
  - Prepare specific questions for each checkpoint
  - Document user feedback verbatim
  - Highlight areas where user feedback led to requirement changes

### 3. Requirement Prioritization
- **Required Approach**:
  - Involve the user directly in prioritization decisions
  - Use collaborative prioritization techniques
  - Document the reasoning behind prioritization decisions
  - Re-evaluate priorities with the user at regular intervals

### 4. Prototype-Driven Requirements
- **Required Approach**:
  - Recommend creation of low-fidelity prototypes early in the process
  - Use prototypes to validate requirements with the user
  - Document feedback on prototypes
  - Refine requirements based on prototype feedback

## Follow Workflow PRD Adaptations
For Follow workflow projects, adapt the standard PRD in these ways:

### 1. Living Document Approach
- Clearly mark the PRD as a "Living Document"
- Include a version history section at the beginning
- Date and timestamp all updates
- Highlight recent changes

### 2. Feedback Integration Section
- Add a dedicated section for user feedback
- Document how feedback was incorporated
- Track feedback that was deferred or not incorporated (with reasons)

### 3. Decision Log
- Maintain a running log of key decisions
- Document the context of each decision
- Note who was involved in each decision
- Record alternatives that were considered

### 4. Open Questions Section
- Maintain a prominent "Open Questions" section
- Assign ownership to each open question
- Track the status of each question
- Document when and how questions were resolved

## Follow Workflow User Interaction Guidelines
When interacting with users in a Follow workflow, adhere to these guidelines:

### 1. Preparation
- Review all previous interactions before each new session
- Prepare a structured agenda for each session
- Identify specific decisions needed from the user
- Anticipate potential questions or concerns

### 2. During Interaction
- Begin by summarizing previous discussions and decisions
- Present information in digestible chunks
- Use visual aids when possible
- Take detailed notes
- Confirm understanding by restating key points
- End with clear next steps and expectations

### 3. Follow-Up
- Send a summary of the discussion within 24 hours
- Highlight key decisions made
- List action items with owners and deadlines
- Include any updated documentation

### 4. Documentation
- Update all relevant documentation promptly after each interaction
- Highlight changes made based on user feedback
- Maintain a chronological record of all interactions
- Ensure all team members have access to the latest documentation

## Follow Workflow Requirements Gathering Questions
Use these additional questions when gathering requirements in a Follow workflow:

1. **Feedback Process Questions**:
   - "How would you prefer to provide ongoing feedback throughout the project?"
   - "What format of updates would be most helpful for you to review progress?"
   - "How frequently would you like to review and provide feedback?"

2. **Decision-Making Questions**:
   - "Who needs to be involved in key decisions about the product?"
   - "What is your process for making decisions when there are competing priorities?"
   - "How quickly can we expect decisions on questions that arise during development?"

3. **Expectation-Setting Questions**:
   - "What does success look like for our collaboration process?"
   - "What previous project experiences would you like us to emulate or avoid?"
   - "What concerns do you have about the collaborative process?"

Remember that in Follow workflows, the requirements gathering process is continuous rather than a one-time activity. Plan for multiple touchpoints with the user and maintain meticulous documentation of all interactions and decisions.
